Useful Git Commands

Create an empty repository in the project folder:

git init

Clone a repository from GitHub and add it to the project folder:

git clone (repo URL)

Retrieve the most recent changes from origin but don't merge:

git fetch

Retrieve the most recent changes from origin and merge:

git pull

Add file changes to staging:

git add (file name)

Remove a file and stop tracking it:

git rm (file_name)

Display the status of modified files:

git status

Display changes in a file compared to the most recent commit:

git diff (file_name)

Commit changes along with a custom message:

git commit -m "(message)"

Switch to a commit in the current branch:

git checkout (commit)

Discard all changes to a commit:

git reset -hard (commit)

Show the history of changes:

git log

Push changes to origin:

git push

Display a list of all branches:

git branch

Make a new branch and switch to it:

git checkout -b (branchname)

Delete a branch:

git branch -d (branchname)

Merge a different branch with your active branch:

git merge (branchname)

OOP in JavaScript

Encapsulation: The programming logic is kept inside the object and only its methods are available to the user to implement the functionality.

Polymorphism: In OOP, this means that different objects can have the same method, but they implement it in their own way.

Inheritance: In OOP, this means that we can take an object that already exists and inherit all its properties and methods. We can then improve on its functionality by adding new properties and methods.

Classes: JavaScript doesn’t use classes (although ECMAScript 6 will support them), but it does use the concept of object prototypes, which can be used as a blueprint for creating other objects.

Constructor Function

JavaScript contains a number of built-in constructor functions such as Object, Array, and Function that can be used to create objects, arrays, and functions instead of literals.

Prototypal Inheritance

We can instantiate an object from a constructor function. The new instantiated object inherits every property and method the constructor has.

The prototype object

Every function has a property called prototype which returns an object. When we instantiate an object using the new operator this object inherits all the properties and methods from the constructor’s prototype object.

  • An easy way of getting the prototype of an object is to use the dunder proto, a fancy name for __proto__.

Own Properties and Prototype Properties

So, what’s the difference between own properties and prototype properties? Prototype properties are shared by every instance of the constructor function. This means that they’ll all have a common property and it will always be the same value. If we create another instance of the constructor, we’ll see that it also inherits a property that has the same value of the original prototype property.

The Prototype Is Live!

The prototype object is live, so if a new property or method is added to the prototype, any instances of it will inherit the new properties and methods automatically, even if that instance has already been created.

What Should the Prototype Be Used For?

The prototype can be used to add any new properties and methods after the constructor function has been defined. It should be used to define any properties that will remain the same for every instance of the constructor.

Public and Private Methods

We can use the concept of variable scope to keep properties private and prevent them from being changed. A getter method can then be used to return their values.
